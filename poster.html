A Comparative Analysis of Ensemble Learning Methods
for Predicting Soccer Match Outcomes

Bagging vs. Boosting vs. Stacking on 25,000+ European Matches
Derrick — Personal Project European Soccer Database (Kaggle) 2026

Introduction

Predicting soccer match outcomes is a challenging three-class classification problem — Home Win, Draw, or Away Win — due to the sport's inherent randomness and the difficulty of capturing pre-match signals that distinguish outcomes.

This project compares three ensemble learning strategies to evaluate which best handles this uncertainty: bagging (Random Forest), boosting (XGBoost), and stacking (Voting Ensemble combining both with Logistic Regression).

The literature reports 55–60% accuracy as the ceiling for three-class soccer prediction using traditional ML methods. Ensemble approaches — particularly gradient-boosted trees — consistently outperform both standalone classifiers and deep learning for this task.

Dataset

European Soccer Database — 25,979 matches across 11 leagues from 2008 to 2016. Includes player FIFA attributes, team tactical profiles, and betting odds from multiple providers.

Feature Engineering

Raw match data alone is insufficient. Six categories of features were engineered from the relational database:

  • Recent Form — Points earned in last 5 matches (3/1/0 for W/D/L) plus rolling goal difference
  • Head-to-Head — Historical win rate between the two teams over last 5 meetings
  • Player Ratings — Average FIFA overall_rating per team using most recent pre-match data
  • Betting Odds — Market-implied probabilities from Bet365, normalized to remove overround
  • Team Tactics — Build-up speed, defence pressure, chance creation from Team_Attributes
  • Contextual — Home advantage indicator, league, and season encodings

All rolling features use only past data (shifted by 1 match) to prevent data leakage. 80/20 stratified train-test split preserves class balance.

Bagging trains independent trees on random subsets (reduces variance). Boosting trains sequential trees that correct prior errors (reduces bias). Stacking combines both strategies for robustness.

Preprocessing Pipeline

  • Connect to SQLite and join Match, Team, Player_Attributes, and Team_Attributes tables
  • Remove duplicates and handle missing values (median imputation for features)
  • Drop non-generalizable columns (match IDs, timestamps, raw player IDs)
  • Engineer all features with temporal ordering to prevent leakage
  • One-hot encode categorical variables (league, season)
  • 80/20 stratified train-test split
  • 5-fold stratified cross-validation during hyperparameter tuning

AUC-ROC Explained

AUC-ROC measures how well the model distinguishes between classes across all probability thresholds. A score of 0.5 = random guessing, 1.0 = perfect separation.

For three-class problems, One-vs-One (OVO) calculates AUC for every pair of classes and averages them.

Conclusion

  • Voting Ensemble achieves the best accuracy (52.2%) and AUC (0.652), while XGBoost leads on F1. The small margins suggest the feature set's prediction ceiling has been reached.
  • Draw prediction is the bottleneck — all models achieve <12% draw recall. Draws lack distinctive pre-match signals compared to decisive outcomes.
  • Results align with published benchmarks — 50–52% accuracy is realistic without in-match event data (xG, shot maps, possession sequences).
  • XGBoost is the most practical choice — 2.4× faster than Random Forest with comparable performance.

Challenges

  • Class imbalance — Home Wins (46%) dominate, biasing all models toward the majority class
  • Draw unpredictability — No strong statistical signatures distinguish draws from wins/losses pre-match
  • Data limitations — Dataset covers 2008–2016; modern tactics and player dynamics differ
  • Feature ceiling — Without in-match event streams, pre-match features hit a natural prediction limit
